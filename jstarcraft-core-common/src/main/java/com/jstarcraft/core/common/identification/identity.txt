主键策略

用户在构建id的时候,使用服务器id作为其中的部分,所有与用户相关的信息的主键,也会使用相同服务器id作为主键标识的一部分 .

水平切割的时候,数据按照服务器id切割,就可以顺利将数据较为均衡的哈希负载到各个服务器.

也较为容易鉴别哪些缓存中的数据需要立即持久化并失效.

为确保以后切割,也为cache提供reset的基本保证
